# Excel to Mobile Image Skill

一个用于将日常周报、工作计划等复杂 Excel 数据表转换成高清晰度、移动端友好图片与美化 Excel 归档的自动化技能。

---

## 适用场景
- 用户需要将复杂的 Excel 数据整理，生成专为手机微信端、钉钉等即时通讯工具查阅的卡片流图片。
- 用户需要过滤 Excel 表格中的多余冗余信息，并对数据按销售人员姓名进行 A-Z 拼音首字母重排。
- 需要将枯燥的传统表格转化为拥有杂志风版面、莫兰迪莫色体系、圆角一致性与高光微阴影的高品质移动端界面。

---

## 核心设计与技术规范（Why it matters）

为了向用户交付极致品质的“反 AI 模板俗气感 (Anti-Slop)”产品，本技能在设计与工程上制定了如下核心规范，执行时请理解其背后的原理：

### 1. 禁用 Emoji，拥抱矢量线稿（视觉品质锁定）
- **规范**：**禁止在标题及主要数据区使用任何表情符号（如 📅、👤、📊、⏰）**，而应使用干净、纤细的 inline SVG 矢量线稿图标。
- **原理**：系统默认 Emoji 在不同操作系统（iOS, Android, Windows）上的渲染风格差异极大，且极易带来低端的 “AI 模板套用感”。SVG 矢量图标则能够保障多端完全一致的高级科技感和杂志级排版细节。

### 2. 几何圆角一致性（Shape Consistency）
- **规范**：大板块容器与核心卡片锁定为 `border-radius: 16px` 大圆角；小型徽章 (Badge)、备注框和序号标签锁定为 `border-radius: 6px` 小圆角。
- **原理**：保持几何比例的严格缩放（大圆角与小圆角的黄金配比），可以让界面呈现出精细的系统级 UI 风格，避免圆角混乱带来的视觉粗糙。

### 3. 莫兰迪雅致配色（Color Consistency）
- **规范**：采用深邃靛蓝（slate-900）作为主色与文字强调色，业务状态徽章采用低饱和度、淡雅冷色调的莫兰迪配色体系（如极浅绿配深绿表示“是”，浅蓝配深蓝表示“技服”）。
- **原理**：高饱和度（如亮红、亮绿）的原色在移动端长图展示时会造成极度刺眼的视觉负担。低饱和度的莫兰迪色系更符合现代人对于高级行政与技术排版的审美追求。

### 4. 灵动岛防遮挡安全区（iOS safe-area）
- **规范**：顶部必须预留至少 `75px` 的安全空白区域。
- **原理**：由于苹果手机等移动端设备具有前置摄像头（灵动岛/刘海屏），顶部若不留足安全区，长图在微信直接打开全屏预览时，主标题极易被物理遮挡。

### 5. 动态高度估算与像素级智能裁剪（性能与防截断）
- **规范**：根据销售分组数及任务行数动态计算逻辑视口高度 `card_height = max(1200, min(15000, estimated_height))`，并在 HTML 底部放置一个隐秘的 RGB `(15, 23, 42)` 实色标记线。截图完成后使用 Python PIL 从底部向上扫描定位该标记进行精准裁剪。
- **原理**：若高度固定（如 9000px），小数据集会留下大片空白且极度浪费 GPU 渲染内存；大数据集则会因容器高度不足被生硬切断。动态计算保障了渲染安全，智能像素裁剪则保证了长图在 Retina 2x 屏幕下完美贴合、绝无空白或内容缺失。

## 核心数据结构

技能默认按以下顺序清洗并重组 Excel 中的字段，抛弃原始表格中的提报时间、发起人等冗余字段：
1. `序号`（脚本根据拼音重排后的顺序自动填充 1, 2, 3...）
2. `所在月周`
3. `销售`（核心排序键）
4. `兵种`
5. `客户名称`
6. `工作类型`
7. `是否支持`
8. `支持人员`
9. `计划时间`
10. `预计工时（h）`
11. `备注`
